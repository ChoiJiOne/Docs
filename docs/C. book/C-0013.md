## 어설픈 변명을 만들지 말고 대안을 제시하라.

변명 대신 대안을 제시하라. 안 된다고 하지 말고 상황을 개선하기 위해 무엇을 할 수 있는지 설명하라. 코드를 버려야 하나? 그들에게 리팩터링의 가치를 가르쳐 줘라. 최선의 방법을 결정하기 위해 프로토타입을 만들 시간이 필요한가? 더 나은 테스트를 도입하거나 재발을 예방하기 위해 자동화를 도입해야 할까? 어쩌면 자원이 더 필요할 수도 있겠다. 부탁하는 것을 어려워하지 말고, 도움이 필요하다는 사실을 인정하라.  

어설픈 변명을 늘어놓기 전에 그 변명꺼리를 없애도록 노력해 보라. 그래도 꼭 해야겠다면 여러분의 고양이에게 먼저 해보라. 아무튼 야옹이가 대신 비난을 받을 수만 있다면야...
